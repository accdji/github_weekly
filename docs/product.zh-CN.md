# 产品设计

[English Version](./product.md)

## 产品定位

GitHub Weekly 不是 GitHub Trending 的镜像站，而是一个基于我们自己历史快照数据构建的周榜产品。

## MVP

第一版聚焦以下能力：

- 从 GitHub 采集候选仓库
- 存储仓库信息和快照历史
- 生成每周热点榜单
- 在网站中展示榜单结果

## 目标用户

- 想发现上升开源项目的开发者
- 需要每周 GitHub 选题的内容创作者
- 跟踪 AI 和开发工具趋势的团队

## 核心页面

### 首页

- 当前周榜 Top 列表
- 分数、本周 star 增量、总 stars
- 语言和更新时间

### 周榜归档

- 历史周榜列表

### 仓库详情页

- 仓库基础信息
- 最新榜单分数
- 最近一段时间的快照历史

## 差异化优势

- 有历史快照数据库，而不只是展示当前结果
- 以每周增长排序，而不是只按总 stars 排序
- 预留中文摘要和分类标签能力
- 后续支持按语言和主题拆分榜单

## 数据策略

### 主数据源

MVP 以 GitHub REST API 为主，因为它结构化且更稳定。

### 补充数据源

后续可以加入 GitHub Trending 作为辅助信号，但不作为唯一数据源。

## 榜单策略

MVP 分数由以下指标构成：

- 7 天 star 增量
- 7 天 fork 增量
- 最近 push 时间带来的活跃度加成

后续可以扩展为主题加权、Trending 加权和新鲜度衰减等机制。
